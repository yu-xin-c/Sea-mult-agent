
# Contributing

感谢您有兴趣参与！

## Before You Start

如果您计划进行重大更改（特别是新功能或设计更改），请在开始工作之前打开问题或讨论。这有助于确保您的工作与项目的方向保持一致。

## Submitting Code

请保持每个拉取请求集中在一个单一的目的。避免在一个PR中混合不相关的更改，因为这会使审查和合并代码变得更加困难。

请尽可能使用 [Conventional Commits](https://www.conventionalcommits.org/) 格式提交消息。这使我们的历史清晰而一致。

在提交代码之前，请运行适当的命令来检查错误并格式化代码。
