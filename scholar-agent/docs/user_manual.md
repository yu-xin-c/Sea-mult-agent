# 🎓 ScholarAgent 智能科研助理平台 - 产品使用手册

## 1. 产品简介

**ScholarAgent** 是一款专为科研工作者和开发者打造的**多智能体（Multi-Agent）任务编排与执行平台**。
该平台能够理解复杂的自然语言指令（如：论文复现、框架选型对比、数据分析等），将宏大任务自动拆解为可执行的 **DAG（有向无环图）工作流**，并调度专门的 Agent 编写代码、配置环境，最终在**安全隔离的 Docker 沙箱**中自动运行并返回结果。

### 核心亮点
*   🧠 **智能任务规划 (Planner)**：基于大模型理解意图，自动生成清晰的子任务依赖关系图 (DAG)。
*   🧑‍💻 **全自动代码生成 (Coder Agent)**：基于字节跳动 **Eino** 框架，调度大模型（如 DeepSeek）生成高质量的执行脚本。
*   📦 **动态沙箱环境 (Docker Sandbox)**：自带联网与完整构建工具链，支持 Agent 自动执行 `git clone` 和 `pip install`，保障主宿主机安全。
*   📊 **可视化交互 (React Flow)**：提供直观的任务流向图，支持节点级别的细粒度监控与触发。

## 2. 环境与依赖要求

在正式运行项目前，请确保您的电脑已安装以下基础环境：
1.  **Docker Desktop**：必须安装并处于**运行中**状态（沙箱执行依赖）。
2.  **Go (1.21+)**：用于运行高并发后端服务。
3.  **Node.js (18+) & npm**：用于运行前端可视化界面。
4.  **大模型 API Key**：默认使用 DeepSeek API，需准备有效的 API Key。

---

## 3. 快速启动指南

### 3.1 启动后端服务 (Go)
后端服务负责处理前端请求、编排 Eino Agent 链以及与 Docker 沙箱交互。

```bash
cd scholar-agent/backend

# (可选) 配置你的大模型 API Key。如果未配置，系统将使用代码中默认的测试 Key
export OPENAI_API_KEY="your-api-key-here"
export OPENAI_BASE_URL="https://api.deepseek.com/v1"
export OPENAI_MODEL_NAME="deepseek-chat"

# 启动服务
go run cmd/api/main.go
```
*启动成功后，后端服务将运行在 `http://localhost:8080`*

### 3.2 启动前端界面 (React)
前端提供了用户输入和 DAG 可视化交互的窗口。

```bash
cd scholar-agent/frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```
*启动成功后，浏览器会自动打开或请手动访问 `http://localhost:5173`*

## 4. 界面功能与操作说明

### 区域 1：任务输入区 (顶部)
*   **功能**：在此输入您想要完成的复杂科研任务或开发需求。
*   **示例输入**：
    > *"假如企业有一批自己的数据，面对市面上有大量 RAG 框架，不知道选用哪个。请检索 RAG 综述，生成对应代码和运行环境，直接运行 A/B 实验得到对应结果。"*
*   **操作**：输入完毕后，点击 **"生成任务计划 (Generate Plan)"** 按钮。此时后端 Planner 会将任务拆解。

### 区域 2：DAG 可视化工作流区 (左侧/中央主体)
*   **功能**：展示任务被拆解后的各个子节点（Node）以及它们的先后依赖关系（Edge）。
*   **操作**：
    *   支持鼠标拖拽画布、滚轮缩放。
    *   **点击任意节点**：即可针对该子任务触发 Agent 执行（例如点击代码生成与执行节点）。

### 区域 3：执行面板与日志区 (右侧侧边栏)
*   **功能**：当您在 DAG 图中点击某个节点后，右侧会滑出执行面板。
*   **操作**：
    *   点击 **"执行当前节点 (Execute Node)"** 按钮，系统会唤醒对应的 Agent 开始工作。
    *   **终端日志**：面板下方会以类似命令行的形式，实时打印 Agent 的思考过程、Eino 节点的流转日志，以及 Docker 沙箱中的标准输出（如 `pip install` 的下载进度或代码运行结果）。

## 5. 典型使用场景演示：自动化框架评估

1.  **第一步：下发任务**。在前端输入框输入评估 RAG 框架的需求，并生成计划。
2.  **第二步：查看计划**。在可视化的 DAG 图中，您会看到如 `文献检索` -> `代码生成` -> `沙箱执行` 等节点流。
3.  **第三步：触发执行**。点击负责执行的节点（如 `Coder_Agent` 相关的代码执行节点），并在右侧面板点击“执行”。
4.  **第四步：观察 Agent 自治工作**：
    *   系统提示 `[System] 正在通过 Eino 框架调用 DeepSeek 模型生成代码...`。
    *   Agent 生成了包含环境搭建和逻辑测试的 Python 脚本。
    *   系统提示 `[System] 正在拉取 Docker 镜像并启动安全沙箱...`。
    *   在日志区，您会看到沙箱内正在自动执行 `git clone` 或 `pip install`。
    *   **最终结果**：沙箱打印出实验对比的输出数据，任务圆满完成。

## 6. 常见问题与排查 (FAQ)

### Q1: 节点执行失败，提示 "Cannot connect to the Docker daemon"？
**原因**：Docker Desktop 未启动。
**解决**：请在系统应用中找到 Docker / Docker Desktop 并打开，等待其图标显示为绿色（Running）后再次重试。

### Q2: 节点执行提示 "代码执行超时(>300s)"？
**原因**：由于沙箱中执行的任务（如克隆大型代码库或安装庞大的依赖，如 PyTorch）耗时较长，超过了系统设定的 5 分钟安全限制。
**解决**：
1. 检查网络环境是否良好，是否能够快速访问 GitHub 或 PyPI。
2. 如需更长执行时间，可由开发者在 `backend/internal/sandbox/docker.go` 中修改超时阈值（当前为 `300 * time.Second`）。

### Q3: 为什么前端点击 "生成计划" 没有反应或提示 404/500 错误？
**原因**：后端服务未启动或端口被占用。
**解决**：请检查终端中 Go 后端的运行状态，确保 `go run cmd/api/main.go` 没有报错，并且 8080 端口未被其他程序占用。可以使用 `lsof -i :8080` 检查。
