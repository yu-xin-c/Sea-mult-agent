# ScholarAgent：智能科研与代码推演多智能体平台 - 架构与目标总结

## 一、 项目目标与背景

**ScholarAgent** 是为2026年（第19届）中国大学生计算机设计大赛设计的全链路智能科研助理与文献沙箱推演系统。

### 核心目标
1. **解决科研痛点**：致力于解决科研人员与开发者在文献分析调研、前沿框架评测、复杂代码复现过程中的繁琐痛点。
2. **突破单体大模型局限**：解决现有大模型工具在处理超长文献时的“大海捞针（Lost in the Middle）”和幻觉问题，以及缺乏复杂任务拆解能力和代码安全执行环境的缺陷。
3. **实现真正的“行动力”（Computer Use）**：从传统的“聊天机器人（Copilot）”向“自主智能体（Autonomous Agent）”演进，赋予AI动态写代码、在安全隔离环境中运行代码并根据报错进行**闭环反思（Self-Reflection）**的能力。

---

## 二、 系统实现架构

项目采用微服务、前后端分离与容器化隔离的现代架构设计，整体分为三大核心层级与一个安全执行层：

### 1. 前端交互层 (Frontend)
- **技术栈**：React + Vite + React Flow
- **功能描述**：
  - 提供类似 ChatGPT 的自然语言对话交互界面。
  - **DAG 实时可视化**：利用 `React Flow` 将后端 Planner 生成的执行计划以可视化的有向无环图（DAG）实时呈现，让多智能体的黑盒协作过程完全透明可见。

### 2. 高并发调度引擎层 (Backend)
- **技术栈**：Go + Gin + Eino (字节跳动开源大模型编排框架)
- **功能描述**：系统的心脏，负责高并发请求处理与智能体调度。
  - **API 网关**：基于 Gin 框架处理前端请求，并提供 WebSocket 流式通信展示思考过程。
  - **动态任务编排 (Planner)**：根据用户自然语言意图，动态拆解任务并生成多智能体协作的有向无环图（DAG）。
  - **多智能体协同 (Multi-Agent System)**：
    - `Librarian Agent`：负责联网查资料或查询本地 RAG 向量库。
    - `Coder Agent`（核心）：基于 Eino 框架调用大模型（如 DeepSeek），根据需求生成 Python 执行代码。
    - `Sandbox Agent`：调度 Docker 容器运行代码，收集并返回标准输出和错误日志。
    - `Data Agent`：负责分析日志和数据，生成评估报告。

### 3. AI 算法微服务层 (AI Services)
- **技术栈**：Python + FastAPI + PyTorch + HuggingFace
- **功能描述**：处理自然语言理解与知识检索的“脏活累活”。
  - **意图识别网关**：前置部署微调后的轻量级 BERT 模型，对用户指令进行精准的意图分类和实体抽取，降低大模型API调用成本。
  - **RAG 检索增强**：结合向量数据库（如 Milvus/Chroma）和 BM25 关键词匹配进行混合检索（Hybrid Search）。通过对 PDF 文献的离线切片（Chunking）与 Embedding，精准召回相关段落喂给大模型，支持跨论文的多跳推理对比。

### 4. 安全沙箱执行层 (Docker Sandbox)
- **技术栈**：Docker Engine + Go Docker SDK
- **功能描述**：系统的护城河与执行核心。
  - **资源隔离**：为大模型生成的代码动态创建隔离的 Docker 容器，严格限制 CPU、内存和网络权限，防止恶意代码破坏宿主机。
  - **闭环反思机制**：捕获容器执行的 `stdout/stderr`，若代码运行报错，则将错误日志作为反馈发回给 `Coder Agent`，触发自我反思与代码自动修正，直至执行成功。

---

## 三、 核心创新点总结

1. **动态 DAG 编排**：告别写死的业务逻辑代码，系统泛化能力强，能适应从“复现论文”到“评测框架”的各类未知复杂任务。
2. **深度集成 Eino 框架**：利用 Graph 将 Prompt 组装、大模型调用和沙箱执行无缝串联，保证了后端的高并发与类型安全。
3. **闭环的 Computer Use**：基于 Docker 沙箱的安全代码执行与 Self-Reflection 机制，代表了 AI 走向自主 Agent 的最前沿方向。
4. **低成本高精度的 RAG**：自建 RAG 向量库取代全文对话，既大幅降低了 API 成本与延迟，又通过精准的片段召回消除了大模型的幻觉问题。
